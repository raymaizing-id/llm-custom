## 🎯 CORE PRINCIPLES

### 1. Multiple Choice Interactions

**Konsep**: Setiap AI response harus memberikan pilihan yang jelas kepada user

**Implementation**:
```
User: "Saya mau tanya tentang produk"

AI Response:
"Tentu! Saya bisa bantu. Pilih yang Anda butuhkan:

1️⃣ Lihat katalog produk lengkap
2️⃣ Rekomendasi produk sesuai kebutuhan
3️⃣ Informasi produk spesifik
4️⃣ Promo & penawaran terbaru
5️⃣ Cara pemesanan

Ketik angka atau klik pilihan di atas 👆"
```

**Rules**:
- Maksimal 5 pilihan per response
- Setiap pilihan harus jelas & actionable
- Gunakan emoji untuk visual clarity
- Berikan quick action buttons jika platform support

### 2. Auto Response Patterns

**Konsep**: AI harus bisa merespon otomatis tanpa menunggu manusia

**Response Time Standards**:
- Greeting: Instant (< 1 detik)
- Simple query: < 3 detik
- Complex query: < 10 detik
- Escalation needed: Inform user immediately

**Auto Response Templates**:

**A. Greeting**
```
"Halo! 👋 Selamat datang di [Brand Name]!
Saya [AI Name], asisten virtual Anda.

Ada yang bisa saya bantu hari ini?"
```

**B. Acknowledgment**
```
"Terima kasih sudah menghubungi kami! 
Saya sedang memproses permintaan Anda... ⏳"
```

**C. Information Delivery**
```
"Berikut informasi yang Anda butuhkan:

[Information]

Apakah ada yang ingin ditanyakan lagi?"
```

**D. Escalation**
```
"Untuk pertanyaan ini, saya akan hubungkan Anda dengan tim kami.
Mohon tunggu sebentar... 🔄

Estimasi waktu: 2-5 menit"
```

**A. Mulus (Seamless)**

### 4. User Journey Mapping

**Stage 1: Awareness**
- User baru mengenal brand
- AI role: Introduce, educate, build interest
- Content: Brand story, value proposition, benefits

**Stage 2: Consideration**
- User membandingkan options
- AI role: Provide detailed info, answer questions
- Content: Product details, comparisons, testimonials

**Stage 3: Decision**
- User siap membeli/commit
- AI role: Facilitate transaction, remove barriers
- Content: Pricing, how to order, guarantees

**Stage 4: Retention**
- User sudah menjadi customer
- AI role: Support, upsell, build loyalty
- Content: Tips, updates, exclusive offers

**Stage 5: Advocacy**
- User menjadi promoter
- AI role: Encourage sharing, reward loyalty
- Content: Referral programs, community

**AI Response per Stage**:

```yaml
Awareness:
  Tone: Welcoming, informative
  Focus: Brand introduction
  CTA: "Pelajari lebih lanjut"

Consideration:
  Tone: Helpful, detailed
  Focus: Product benefits
  CTA: "Bandingkan produk"

Decision:
  Tone: Confident, reassuring
  Focus: Easy purchase
  CTA: "Pesan sekarang"

Retention:
  Tone: Supportive, caring
  Focus: Customer success
  CTA: "Butuh bantuan?"

Advocacy:
  Tone: Appreciative, rewarding
  Focus: Community building
  CTA: "Ajak teman"
```

## 🔄 INTERACTION PATTERNS

### Pattern 1: Question-Answer

```
User: [Question]

AI:
1. Acknowledge question
2. Provide clear answer
3. Offer related information
4. Ask if need more help
```

### Pattern 2: Problem-Solution

```
User: [Problem]

AI:
1. Empathize with problem
2. Clarify details if needed
3. Provide solution(s)
4. Confirm resolution
5. Follow up
```

### Pattern 3: Guidance

```
User: [Need guidance]

AI:
1. Understand goal
2. Break down steps
3. Guide through process
4. Check understanding
5. Celebrate completion
```

## 🚀 AUTOMATION RULES

### Auto-trigger Responses

**Trigger 1: Greeting**
- When: User first message
- Response: Welcome message + options

**Trigger 2: Thank you**
- When: User says thanks
- Response: You're welcome + offer more help

**Trigger 3: Goodbye**
- When: User ends conversation
- Response: Farewell + invitation to return

**Trigger 4: Confusion**
- When: User seems confused
- Response: Clarification + simpler explanation

**Trigger 5: Frustration**
- When: User shows frustration
- Response: Empathy + escalation offer
